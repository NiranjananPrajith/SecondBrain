# Vault Organization Guidelines

**This file is a quick reference only.** The comprehensive organizing guide is at `memory/vault-organizing-guide.md`.

## File Naming

| Type | Format | Example |
|------|--------|---------|
| Dated | `YYYY-MM-DD-topic.md` | `2026-04-13-rvr-planning.md` |
| General | `CamelCase.md` | `RedVelvetRenders.md` |

## Folder Structure

- Top-level: `journal/`, `memory/`, `inbox/`, `interests/`, `meetings/`, `issues/`, `people/`, `projects/`
- Folder names: `lowercase-hyphenated/`
- Max 2–3 levels deep

## Frontmatter

```yaml
---
title: Title Case
created: YYYY-MM-DD
tags: [tag1, tag2]
---
```

## Research Structure

```
research/
  queries/      # inputs (briefs, questions)
  reports/      # outputs (AI results, analysis)
  synthesis/    # (optional) distilled insights
```

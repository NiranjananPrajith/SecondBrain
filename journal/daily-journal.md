# Daily Journal

Track your day here. Start each morning by asking "What should I work on today?" — I'll review the relevant files and give you an overview.

## Rules

1. **Every update goes to both journal AND parent document.** If I update `PaidAdSprint.md`, I also log it here.
2. **Timestamps on all updates.** Use `- [YYYY-MM-DD]` format so the journal is scannable.
3. **Link parent documents in each update.** Every time a parent doc is modified, note it in the update entry.
4. **End of day summary is mandatory.** Even if nothing got done — write it down. The carry-forward tells tomorrow's brief what to pick up.
5. **Write for yourself, not for an audience.** Raw, honest, in-the-moment. It's a working document.
6. **No fluff.** If a section has nothing in it, skip it or leave a single `-` so it's clear you considered it.
